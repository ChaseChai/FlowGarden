## 1. 产品概述

### 1.1 一句话产品

> **镜园是一款状态感知的游戏化健康伴侣：一枚戒指读懂你的睡眠与专注，数据长成一座花园，数字分身与你同呼吸。**

### 1.2 产品定位回顾

- **品类**：状态感知健康伴侣（State-aware Wellness Companion），不做"专注 APP"或"睡眠 APP"
- **市场空缺**：数据真实度 × 表达温度的右上象限空白（详见 `镜园_竞品分析与产品定位.md`）
- **三大支柱**：① 戒指真实感知（vs Forest 手动打卡）② 世界化表达（vs Oura 分数报告）③ 镜像数字人（vs Replika 无数据陪伴）

### 1.3 目标用户

| 层级 | 画像 | 核心诉求 |
|------|------|----------|
| 核心层 | 18-25 岁学生/考研党/自习人群 | 专注陪伴，拒绝惩罚式打卡 |
| 增长层 | 22-30 岁年轻职场人/创作者 | 搞懂"为什么睡 8 小时还累" |
| 延展层 | ADHD/注意力困难群体 | 低压力、被理解的状态管理 |

### 1.4 成功指标（Hackathon 维度）

| 指标 | 目标 |
|------|------|
| Demo 理解成本 | 评委 10 秒内看懂规则 |
| 核心闭环 | 现场完整跑通"睡眠报告 → 专注生长 → 分心落叶 → 休息互动" |
| 视觉记忆点 | ≥1 个"想截图"的画面（月光花园/数字人打哈欠） |
| 技术可信度 | 睡眠分期/专注判定可现场验证，非纯动画 |

### 3.2 P0 功能详述

#### F1 戒指连接与数据接入

**需求描述**：APP/网页启动后通过 BLE 连接戒指，建立数据通道。

**输入**：
- 戒指 MAC 地址（配置项）
- `ring_sound.py` SDK 能力：`RingSoundClient`、`wait_sensor_data()`（0x0605）、`wait_sensor_key_single_press_event()`（0x0704）、`wait_sensor_key_double_press_event()`（0x0703）、`wait_sensor_gesture_event()`（0x0702）

**处理**：
1. 启动 → 扫描/直连戒指 MAC
2. 订阅 IMU 批量数据流与按键/手势事件
3. 连接状态心跳检测，断连自动重连（指数退避，最多 5 次）

**输出**：统一事件总线 `RingEvent = {type: imu_batch | tap | double_tap | gesture | disconnect, payload, timestamp}`

**验收标准**：
- 冷启动到首个 IMU 数据包 ≤ 10s
- 断连后 30s 内自动恢复，UI 有连接状态指示

**已知约束**（来自 SDK 文档）：
- 实时 IMU 需设备处于手势模式且 `start_sensor_report()` 成功；录音模式下 0x0601 返回错误码 2
- **降级方案**：戒指 IMU 不可用时，允许切到"演示数据源"（预录数据回放），保证 Demo 可跑

#### F2 睡眠监测与分期

**需求描述**：夜间佩戴戒指，基于 IMU 体动进行 actigraphy 睡眠分期。

**算法**（医学经典方法论，与 Oura 同源）：

```python
WINDOW = 60s（1 分钟 epoch，医学标准）
activity = 该 epoch 内 |accel| 超过阈值的采样点数

activity == 0      → deep_sleep（深睡）
1 <= activity <= 2 → light_sleep（浅睡）
3 <= activity <= 8 → rem（REM，手指微动——戒指独特优势）
activity > 8       → awake（清醒/翻身）
```

**衍生指标**：
- 总睡眠时长、入睡耗时（躺下→首次深睡）
- 深睡比例、睡眠周期数（深/浅交替 ≈ 90min/周期）
- 翻身次数（大体动）、夜间觉醒次数

**验收标准**：
- 预录 IMU 数据回放时，输出分期序列与指标
- Demo 现场可用 3 分钟模拟数据演示分期变化

#### F3 晨间睡眠报告

**需求描述**：双击戒指"起床"，生成睡眠报告，数字人口播 + 可视化呈现。

**内容结构**：
1. 总结分（0-100，深睡比例 40% + 时长 30% + 周期完整性 30%）
2. 关键指标卡片（深睡时长/比例、夜醒次数、入睡耗时）
3. 睡眠结构曲线（深/浅/REM/醒 时间轴）
4. 数字人一句话解读（LLM 生成，模板兜底）：
   > "你睡了 7 小时 42 分，深睡只有 58 分钟——比上周平均少 20%。这就是你今天累的原因。"

**花园联动**：花朵带露珠，露珠数量/大小 ∝ 深睡比例。

**验收标准**：从双击到完整报告呈现 ≤ 3s；LLM 不可用时模板文案兜底。

#### F4 专注模式

**需求描述**：双击戒指进入专注，IMU 判定专注/分心，驱动花园生长或落叶。

**判定算法**（5s 滑窗，1s 步长）：

```python
features = {
    variance: 三轴加速度合成方差,
    zero_crossing: 过零率,
}
variance < T1 且 zero_crossing < T2 → deep_focus（稀有花进度+2）
variance < T1                      → focus（普通生长）
否则                                → distracted（不落叶，仅暂停生长）
```

**拍击 = 主动标记分心（诚实记账）**：
- 触发落叶动画 + 记录分心时刻（时间戳 + 当时长）
- 累计诚实标记 → 成就进度（诚实之泉，F13）
- **设计原则：惩罚最小化**——检测到的分心只暂停生长，只有"诚实承认"才有视觉反馈，且反馈是奖励性的

**双击 = 结束专注** → 结算本次 session（时长、深度比例、植物入册）。

**验收标准**：现场演示"静止 30s 发芽 → 晃动 10s 生长暂停 → 拍击落叶动画"全链路 ≤ 90s。

#### F5 花园渲染与状态机

**需求描述**：主界面花园，艺术氛围层（不做 L-system 精细生成），状态驱动切换。

**实现方案**：
- **技术栈**：Three.js（或 PixiJS 2D 备选）+ 预渲染美术资源 + 粒子层（花瓣/落叶/萤火虫/露珠）+ 光照层（晨光/正午/黄昏/月光）
- **植物生长**：骨骼缩放动画（种子→芽→株→花 4 档预制模型/贴图），不做算法生成
- **天气系统**：晴/多云/雨/月光 4 态，gsap 渐变过渡（2s）

**状态映射表**：

| 状态 | 天气 | 光照 | 粒子 |
|------|------|------|------|
| 深度心流 | 晴 | 金色暖阳 | 金色花粉 |
| 浅度专注 | 多云 | 柔白 | 微风飘絮 |
| 分心标记 | 雨（10s 短促） | 微暗 | 落叶/花瓣飘零 |
| 疲惫 | 黄昏 | 暖橙 | 长影 |
| 休息/睡眠 | 月夜 | 蓝调月光 | 萤火虫/露珠 |

**验收标准**：5 种状态现场可切换，过渡自然；画面达到"想截图"水准。

#### F6 数字分身

**需求描述**：住在花园里的数字人，映射用户状态，承担报告与互动的"嘴和脸"。

**形象方案**（按工期二选一）：
- 首选：Live2D 模型（2D，轻量，表情丰富，加载快）
- 备选：VRM + three-vrm（3D，骨骼动画）

**P0 动作集**（8 个预置动画状态机）：

| 动作 | 触发 |
|------|------|
| 静修冥想 | 用户专注中 |
| 修剪/看书 | 用户浅专注 |
| 回头看 | 检测到分心 |
| 打哈欠/揉眼 | 晨间报告显示睡眠不足 |
| 击掌/打招呼 | 拍击互动 |
| 跳舞庆祝 | 成就达成 |
| 守夜坐姿 | 睡眠监测中 |
| 门口等待 | ≥3 天未互动（怀念态） |

**验收标准**：晨间报告由数字人"说出"（TTS 或文字气泡）；至少 5 个动作现场可触发。

### 3.3 P1 功能详述（简版）

#### F7 休息模式互动
- 双击切换专注/休息；休息模式下手势 `wave`=召唤分身走近、`rotate_front`=种新种子、`rotate_back`=切换天气
- 拍击 = 击掌/触发小动作

#### F8 入睡仪式
- 进入睡眠模式：屏幕渐暗（5min），数字人 4-7-8 呼吸引导动画（环形光 4s 扩张/7s 停留/8s 收缩）
- IMU 检测逐渐静止（入睡信号）→ 自动熄屏进入 F2 监测

#### F9 语音对话
- 戒指录音 → `receive_auto_audio_file()` → SDK 解码 WAV → ASR（Whisper API 或本地 vosk）→ LLM 意图分类（对话/指令/情感）→ 数字人 TTS 回应
- 降级：无 ASR 时支持预置指令按键菜单

#### F10 数据页
- 心流曲线（日/周热力图）、睡眠趋势（周）、分心时刻回放列表（"14:32 你标记了一次分心"）

### 3.4 P2 功能（有时间再做）

- F11 关联洞察：`corr(deep_sleep_yesterday, distraction_today)` 周报卡片
- F12 多画风皮肤：水彩（默认）/像素/国风 3 套资源切换
- F13 成就：诚实之泉（10 次诚实标记）、极光之夜（连续 7 天深度专注 ≥2h）、月度织锦回顾

### 4.3 关键协议（WS 消息）

```json

// 睡眠引擎 → 前端
```

### 4.4 技术选型与理由

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 前端渲染 | Three.js | 团队熟悉度 + 粒子/光照生态 |
| 数字人 | Live2D 优先 | 比 VRM 轻量、表情丰富、48h 可落地 |
| 桥接 | Python + WebSocket | 复用 ring_sound.py，零改动 SDK |
| ASR/LLM | API 优先，模板兜底 | 现场网络不可控，必须有离线降级 |
| 数据源 | 真戒指 + 预录回放双通道 | **Demo 不死是第一原则** |

## 5. 非功能需求

| 类别 | 需求 |
|------|------|
| 性能 | 花园场景 60fps（中端笔记本）；WS 延迟 <100ms |

| 演示可靠性 | 全功能支持预录数据回放模式（DEMO_MODE=1） |
| 隐私 | 睡眠/语音数据本地处理；语音仅在上传 ASR 时离机，可关闭 |
| 兼容 | Windows 10/11 + Chrome/Edge；BLE 依赖 bleak |
