## 1. 设计总纲

### 1.1 V3 定位对画风的核心影响

| V3 变化 | 对画风的影响 |
|---------|-------------|
| 融合 Nexday → Nexi 是"有数据的秘书" | 数字人不能太萌系/宠物化，需要**可信赖感** |
| 优先级倒转 → 前端视觉是工期重心 | 画风决策必须 **48h 可实现** |
| 赛道新增 03 灵光（Vibe） | 需要**强氛围感**，不依赖复杂交互 |
| "绚丽花园"替代"治愈花园" | 色彩饱和度和光影戏剧性需要提升 |
| 三轴展演（数据轴+手势轴+秘书轴） | 花园需要能被"看见"在呼吸，不能只是静态背景 |

### 1.2 核心设计原则

1. **Demo 不死是第一原则** → 只用可靠的技术方案（Three.js + gsap + Live2D）
2. **48h 出"想截图"的画面** → 氛围靠几何块面和色彩，不靠大量手工美术
3. **严肃的数据，温柔的表达** → 数据映射到块面色阶/露珠面数/光束层级，不出现在 UI 上
4. **滋养非惩罚** → 分心 = 暂停生长+10s落叶，不是枯萎/红色警告
5. **花园是活的世界，不是一张画** → 块面重组、色阶流动、几何生长

### 1.3 核心视觉哲学：塞尚的几何诚实

> **「用圆柱体、球体、圆锥体来表现自然」—— 塞尚，1897**
>
> 塞尚用几何体块画苹果，不是因为苹果是方的，而是因为**几何是最诚实的方式**——每一个色块都是一个独立的观察，每一面都是一次真实的判断。
>
> 镜园的视觉哲学继承于此：**你的深睡是露珠的色块面数，你的专注是花瓣亮面的明度，你的分心是叶片暗面的深度。我们不是把数据画成花园——我们让数据以几何的诚实，自己构造成花园。**
>
> 技法落地：**以塞尚的块面几何为造型逻辑，以 Firewatch 的纯色块为色彩策略，以 Ori 的辉光粒子为氛围技法，以 Journey 的无 UI 沉浸为体验目标。**

### 1.4 一句话定调（v1.3 升级）

> **不是"用花园来表达数据"，而是"数据以几何的诚实自己构造成一座花园"。**

### 2.3 关键洞察

> 全球最受赞誉的"艺术游戏"有一个共同特征：**它们不是把美术当作装饰，而是把美术当作叙事的承载媒介**。Gris 用色彩解锁来讲述走出抑郁，Journey 用远处的光来引导前行，Ori 用辉光来区分安全与危险。
>
> **镜园更进一步**：我们不仅让美术承载叙事——我们让美术的**造型逻辑本身**成为数据的隐喻。塞尚用几何块面画苹果，因为他相信几何是最诚实的方式；镜园用几何块面画花园，因为花园里的每一片花瓣、每一颗露珠都来自真实的身体数据。**几何的诚实 = 数据的诚实。**

## 3. 主推画风方案：塞尚花园（俯瞰式算法花园）

> **花园形态借鉴 Pollinator Pathmaker 的算法布局 + 塞尚的块面几何造型。**
>
> 不是从正面"看花园"，而是从**斜 45° 俯瞰整座花园**——如同 Pollinator 让用户看到传粉者眼中的花园，镜园让用户看到"数据眼中的自己"。

### 3.1 方案定位

> **Pollinator 的算法布局 × 塞尚的块面造型 × 真实天气数据 × 戒指实时感知 = 镜园的「俯瞰式算法花园」**

### 3.2 花园形态：俯瞰式网格布局

#### 为什么是俯瞰而非正面？

| 正面视角（旧方案） | 俯瞰视角（新方案 ✅） |
|-------------------|---------------------|
| 只能看到前方几株植物 | **整座花园一览无余——数据全景** |
| 纵深遮挡植物 | **无遮挡，每株植物可见** |
| 像"壁纸"——静态画面感 | **像"地图"——可探索、可规划** |
| 难以表达布局逻辑 | **天然适配网格化算法布局** |

│       雨天=冷蓝 α0.2 + 雨滴粒子 / 雪=白色 α0.3)     │
├──────────────────────────────────────────────────┤
│  L4: 专注轨迹层 — 半透明金色多边形路径                │  Three.js
│      (宽度∝专注深度 / 金色度∝心流占比)               │  Line/Plane

│      植物 = 硬边几何色块(见§4)                       │  flatShading
├──────────────────────────────────────────────────┤
│  L2: 粒子氛围层 — 天气驱动粒子                       │  Three.js
│      (晴天=花粉 ◆ / 阴天=飘絮 ■ / 雨天=雨滴 ◢ /     │  InstancedMesh
│       雪=雪花 ❖ / 月夜=萤火虫 ✦)                    │
├──────────────────────────────────────────────────┤
│  L1: 地面层 — 大块多边形色块地面 + 网格线              │  Three.js
│      (暖调大地色系 / 区块解锁=色彩变亮)               │  Plane Mesh
├──────────────────────────────────────────────────┤
│  L0: Nexi 层 — Live2D 数字人 (边框右下)              │  独立 canvas
└──────────────────────────────────────────────────┘
```

**关键视角参数**：
- 相机：`OrthographicCamera` 或 `PerspectiveCamera (fov: 45°, position: [0, 12, 16])`
- 角度：斜 45° 俯瞰（既能看到植物顶部色块，也能看到侧面）
- 旋转：用户可轻拖旋转 ±15°（非自由旋转，保持"地图感"）
- 缩放：滚轮缩放 0.8x - 2x

### 3.8 硬边层叠光束 — 核心视觉签名

> 心流状态下的多边光束是镜园最有辨识度的视觉特征。不同于 Ori 的柔和径向渐变，镜园的光束由**逐级增亮的多边形层叠**构成——每一层都是独立色块，层与层之间是硬边分界。

**美术史渊源**：塞尚画苹果时，亮面和暗面的交界处没有渐变过渡——就是两块不同颜色的多边形紧挨在一起。镜园的光束继承这个逻辑：从 `#FFF8E1`（最亮层）到透明，中间是 5 层逐级变暗的硬边多边形。

**技法**：
- Three.js：`Mesh` 多个半透明平面多边形，沿光束方向排列；每面使用独立纯色材质
- 简单方案：Canvas 2D 绘制多边色块叠加层 + CSS `mix-blend-mode: screen`
- 层级数量：5 层
- 光束角度：从画面左上角向右下倾斜约 30-45°
- 光束色阶：`#FFF8E1` → `#F5D76E` → `#F0C27A` → `#D4A96A` → 透明
- 心流状态：5 层全显
- 浅专注：2-3 层
- 分心/疲惫/月夜：0 层（消失）

### 3.9 备选默认：晨曦花园（柔和向）

> 如果团队评估后认为评审更偏好柔和治愈感，可切换为晨曦花园作为默认皮肤。两者共享相同的六层架构、天气系统、粒子类型和 Nexi 动作——区别仅在于**材质渲染方式**（flat vs smooth）和**色彩饱和度**。详见 §6.1。

### 3.10 算法花园布局引擎（借鉴 Pollinator Pathmaker）

> **灵感来源**：[Pollinator Pathmaker](https://pollinator.art) — Alexandra Daisy Ginsberg 的算法艺术项目。用规则引擎替代人类审美，让算法"为传粉者设计花园"。
>
> **镜园映射**：Pollinator 的算法哲学 → 镜园的"数据驱动花园布局"。不是人类设计师摆放每一株植物，而是**用户的睡眠/专注数据，通过算法规则，自己决定花园的样子**。

#### 3.10.1 核心哲学对照

| Pollinator Pathmaker | 镜园 FlowGarden |
|---------------------|-----------------|
| **算法同理心** = 为传粉者设计 | **算法同理心** = 为用户的身心健康设计 |
| 输入：光照、暴露、花园尺寸 | 输入：深睡比例、专注深度、分心频率、连续自律天数 |
| 植物调色板：区域物种库（82种） | 植物调色板：5 种花草 × 4 生长阶段 + 1 稀有花 |
| 规则：不同嘴型→不同花型、花期覆盖四季 | 规则：生长速度 ∝ 专注深度、稀有花概率 ∝ 一致性 |
| 传粉者觅食路径 = 最优飞行轨迹 | **专注轨迹** = 用户的分心/心流时空分布 |
| 季节滑块 = 花期随月份变化 | **5 态自动切换** = 花园随 24h 身心节律变化 |
| 一次性生成 → 打印种植方案 | **连续实时生长** = 花园是活的，每时每刻在变化 |
| 反人类审美——"不自然的花园，为自然而设计" | **反仪表盘——"不像数据报告的花园，为数据而生长"** |

• "不同花型适配不同状态" — 每种花有独特触发条件

• "避免人造品种" — 不使用看起来"太完美"的对称花型，保留自然不规则感

#### 3.10.4 专注轨迹 (Focus Traplines)

Pollinator 的核心洞察之一是蜜蜂的 **trapline**（最优觅食路径）。镜园将其映射为**专注轨迹**：

```
蜜蜂 trapline:                     镜园 专注轨迹:
─────────────                     ─────────────
蜜蜂记住每朵花的位置              系统记住每次专注 session 的时间/时长
每天沿最优路径飞行                 每天生成"专注热力轨迹"叠加在花园上
路径可视化在雷达图中               轨迹可视化在花园地面上（金色多边形路径）

实现:
  花园地面上叠加一层半透明多边形路径
  路径的宽度 ∝ 专注深度
  路径的金色度 ∝ 心流占比
  一周的轨迹叠加 → 看到"你的花园里哪些地方走过最多"
```

**答辩金句**：
> "蜜蜂在花丛中留下飞行轨迹——你在花园里留下专注的轨迹。一周后回看，你能看到自己的心流在花园里踏出了什么样的路径。"

#### 3.10.5 每日花园快照 vs 累积生长

| 设计决策 | Pollinator 的做法 | 镜园的做法 |
|----------|------------------|-----------|
| 生成频率 | **一次性生成**，输入参数→输出固定方案 | **每日评估 + 实时微调** |
| 植物留存 | 方案不变，植物会真实生长 | **累积生长**——长出来的植物永久保留 |
| 布局变化 | 不变 | 每天早上 Nexi 根据昨晚数据决定：新增植物位置、解锁新区域 |
| 重置机制 | "Create new artwork" 手动重来 | 无重置——花园是你生活的**不可逆累积** |

**设计理由**：镜园的花园不能"重来"，因为你的生活不能重来。那些"分心多所以长得慢"的植物，正是你真实状态的见证——它们比茂密的花园更有意义。

#### 3.10.6 反仪表盘哲学

> Pollinator 的核心贡献：**用算法消除人类审美偏见，设计出"人类不会设计但自然界需要"的花园。**
>
> 镜园的核心贡献：**用数据规则消除仪表盘思维，生长出"人类不会画但身体数据需要"的花园。**
>
> 不是「把睡眠数据可视化为花园」，而是「睡眠数据被算法规则转化为花园的密度/生长/稀有度」。用户不需要理解规则——看一眼花园的繁茂程度就知道自己最近过得好不好。这就是 Pollinator 教给我们的：**算法不是用来展示数据的，是用来替数据做设计决策的。**

## 4. 花卉/植物视觉规范（硬边几何版）

### 4.1 造型原则

> **每一片花瓣、每一片叶子都由硬边多边形色块构成。亮面/中间调/暗面明确分界——没有渐变过渡，只有色块的诚实。**

### 4.2 形态方案

#### 方案 A：硬边 Low Poly 几何体（推荐，默认）

| 维度 | 说明 |
|------|------|
| 参考 | 塞尚静物 + 当代硬边 Low Poly 插画 |
| 实现 | Blender 快速建模 5-6 种植物，每种 4 档形态，导出 glTF；**确保每个面是独立纯色顶点色** |
| 材质 | `MeshStandardMaterial({ flatShading: true, roughness: 0.6, metalness: 0 })` |
| 光照 | 每个面由法线方向决定亮度——面朝光源=亮面色块，背离光源=暗面色块，侧面=中间调色块 |
| 优点 | Three.js 直接加载，硬边几何感天然正确，体积小 |
| 缺点 | 需要基本的 3D 建模能力，面数需手动控制（每株 50-150 面） |

#### 方案 B：Sprite 硬边序列帧（备选）

| 维度 | 说明 |
|------|------|
| 参考 | 手绘硬边色块插画 |
| 实现 | 手绘 4 帧 PNG，每帧用纯色块（无渐变），billboard 朝向相机 |
| 优点 | 零建模成本，色块画风完全可控 |
| 缺点 | 无法旋转视角，层次感不如 3D |

### 4.3 植物类型与生长阶段（几何版）

```
阶段 0: 🌰 种子 — 小多面体（6-8面）土堆，微微颤动（色块微移）
阶段 1: 🌱 嫩芽 — 三角形叶片（3-5面）破土，亮面 #B4D89A / 暗面 #5B8C3A
阶段 2: 🌿 植株 — 面数递增（20-40面），茎干=细长三角柱，叶片=五边/六边形色块组
阶段 3: 🌸 开花 — 面数达到峰值（60-120面），花瓣=多边形色块簇，每面纯色
阶段 S: 🌷 稀有花 — 额外 20-40 面 + 每面微自发光 + 金色边缘面

生长驱动: 专注 session 时长 → growProgress 0→1 → 顶点沿法线方向挤出（新面生成）
生长动画: ease-out，面数以"块面增殖"方式增长——不是平滑缩放，而是新色块逐个出现
```

### 4.5 露珠设计（多面体宝石版）

| 维度 | 说明 |
|------|------|
| 出现时机 | 晨间简报（双击起床后），只在花朵亮面上 |
| 数量 | ∝ 昨晚深睡比例——深睡越多，**露珠面数越多**（深睡 40%→12面体，深睡 70%→20面体） |

| 隐喻 | **"深睡的质量 = 露珠的复杂度"**——越多面体越接近完美的宝石

### 5.2 块面化 Nexi 方案（塞尚风格专属）

- 绘制标准 Nexi 插画 → 将插画的每个明暗过渡区转为硬边多边形色块（可用滤镜预处理）
- Live2D 的纹理贴图使用块面化版本
- 动画时色块随骨骼运动重新排列——产生独特的"几何生命感"

| 身体部位 | 块面化处理 |
|----------|-----------|
| 脸部 | 亮面（额头/鼻梁/颧骨）= 3-4 个肤色亮块；暗面（脸颊阴影/下颌）= 2-3 个肤色暗块 |
| 眼睛 | 眼白=白色块，瞳孔=深棕色菱形块，高光=白色小三角 |
| 头发 | 深棕色多边块簇（12-18 面），亮面 `#8B6914` / 暗面 `#3D2A0A` |
| 围巾 | 苔绿多边块，折叠处=暗面色块 `#2D4F22` / 展开处=亮面色块 `#6DA85D` |
| 上衣 | 米色大面积平涂色块，褶皱处=暗面小块 `#D4C5B0` |

## 6. 四大皮肤方案

### 6.1 🍎 塞尚花园（默认皮肤）

> **艺术溯源**：后印象派塞尚 (Cézanne) → 立体主义块面 → 当代硬边 Low Poly 写实动画
>
> 塞尚说「用圆柱体、球体、圆锥体来表现自然」——镜园说「用几何色块来表现你的身体数据」。

#### 风格定位

| 维度 | 说明 |
|------|------|
| **学名** | 块面几何厚涂动画 / 写实向 Low Poly 二维插画动画 |
| **英文关键词** | `geometric block painting animation, hard edge flat color realistic illustration` |
| **风格参考** | 塞尚静物油画 + 当代硬边 Low Poly 写实动画短片 |
| **核心特征** | 画面全部拆解为**边界清晰的硬边几何色块**，没有柔和渐变，每一块明暗都是独立多边形，同时保留写实物象结构 |
| **与纯 Low Poly 3D 的区别** | 这是**二维/2.5D插画动画**，用手绘硬边块面模拟体积，非三维模型渲染 |

#### 为什么塞尚风格是默认选择？

| 维度 | 契合点 |
|------|--------|
| **"严肃的数据"** | 几何块面天然带有理性、精确、可量化的视觉联想——正是"戒指真实数据"的视觉隐喻 |
| **"温柔的表达"** | 暖色调的色块拼接可以同时做到结构清晰但不冰冷——像塞尚的苹果一样温暖 |
| **独特性** | 全球健康/专注类 App 中没有一款用这个风格——评审会记住 |
| **艺术格调** | 塞尚是"现代绘画之父"，引用美术史比引用游戏更有文化深度和答辩叙事 |
| **48h 可行** | `flatShading: true` 就是核心技术——Three.js 原生支持，不增加开发复杂度 |
| **"数据构成生命"的隐喻** | 花园的每一片花瓣、每一缕光都由几何色块构成——正如你的每一个深睡分钟、每一段心流构成了今天的你 |

#### 画风实现对照

| 画面元素 | 塞尚花园中的表现 |
|----------|-----------------|
| **植物** | 花瓣=多边形硬边色块拼接，叶片=三角形/四边形色块在光照下呈现不同亮度 |
| **天空** | 大块面的色域平涂（辉光不是渐变，而是逐级亮度的多边形层叠） |
| **光影** | 不是 soft shadow，而是**硬边多边形明暗块**——亮面/暗面/中间调明确分界 |
| **粒子** | 硬边几何小方块（■ ◆ ▲）代替圆形软粒子——花粉=金色小三角群，落叶=棕色四边形碎片 |
| **露珠** | 多面体宝石质感——半透明硬边菱形块 + 高光面，面数 ∝ 深睡比例 |
| **地面/路径** | 不规则多边形色块拼接，低饱和暖色调 |
| **Nexi** | 角色用块面拆解——脸部=肤色多边色块，头发=深棕色几何块，衣服=大面积平涂色块 |
| **硬边层叠光束** | 不再是从一点扩散的柔和渐变，而是**逐级增亮的多边形层叠光束** |

#### 色彩策略

```
塞尚花园色板（比晨曦花园更浓郁、更"油画感"）：

🌅 天空色块:
  晨光: #F2D5A3 / #E8C48A / #D4A96A （暖杏→蜂蜜→琥珀）
  正午: #E8E0D5 / #C9D5E0 / #A8B8C8 （奶油→灰蓝→石板蓝）
  黄昏: #D4784A / #B8553A / #8B3A2A （陶土橙→焦赭→深棕红）
  月夜: #3A4A6B / #2A3552 / #1A2038 （钴蓝→靛蓝→深墨蓝）

🌿 植物色块（亮面/中间调/暗面三阶）:
  亮面:  #8BBA7A → #6DA85D → #4E8A3E （黄绿→草绿→深绿）

#### 实现技法

| 技法 | 说明 |
|------|------|
| **硬边渲染** | Three.js `MeshStandardMaterial({ flatShading: true })` — 每个面独立计算光照，面与面之间硬边分界 |
| **色阶量化后处理** | 可选：将渲染结果量化为 5 级色阶 → 强化硬边感 |
| **植物建模** | Low Poly 几何体，每个面是纯色顶点色，不设渐变纹理 |
| **天空** | 大块多边形面片层叠，每个面片独立纯色 |
| **粒子** | `InstancedMesh` with Tetrahedron/Box geometry — 硬边小块 |
| **Nexi** | Live2D 纹理预处理为块面化版本（见 §5.2） |
| **光束** | 5 层多边形平面 Mesh，逐级亮色（见 §3.5） |

#### 与 AI 绘画/短视频平台的契合

- `geometric block painting style, hard edge flat color, Cézanne inspired`
- `low poly realistic illustration, polygonal color blocks, warm earth tones`
- `硬边块面油画动画、分色块写实插画`

#### 答辩叙事

> "塞尚用几何体块画苹果——我们用它来画你的生活。每一片花瓣、每一缕光、每一颗露珠，都是由独立的多边形色块构成。这些色块不是装饰，它们是数据：深睡的分钟数决定露珠有多少个面，专注的深度决定花瓣亮面有多亮。**严肃的数据，用几何的诚实来表达。**"

## 8. 动效与交互反馈

### 8.1 动效总原则

| ✅ 要 | ❌ 不要 |
|------|--------|
| gsap 2s 缓动过渡 | 线性 0.3s 硬切 |
| 自然物理曲线（ease-out） | 弹跳过度（spring） |
| 微粒子的缓慢漂浮 | 满屏爆炸粒子 |
| 呼吸式的明暗脉动（~4s 周期） | 闪烁/高频变化 |

### 8.2 具体动效规范

| 事件 | 动效 | 时长 | 曲线 |
|------|------|------|------|
| 状态切换（任一） | 全部 5 层同步渐变 | 2s | `power2.inOut` |
| 拍击诚实标记 | 一片叶子从植物飘落 + 微弱涟漪扩散 | 1.5s | `ease-out` |
| 植物生长（4档切换） | 骨骼缩放 + 轻微弹性回弹 | 1s | `ease-out`+`back(1.2)` |
| 新种子种植 | 种子落下→小土堆→芽破土→两片叶展开 | 3s | `ease-out` |
| Nexi 动作切换 | Live2D 混合过渡 | 300ms | `ease` |
| 对话气泡出现 | 淡入 + 微上浮 | 0.5s | `ease-out` |
| 露珠凝聚 | scale 0→1 + opacity | 2s | `ease-out` |
| 月光呼吸（数据轴） | 月光亮度随 IMU 体动起伏 | 实时 | 直接映射 |
| 喝彩花瓣雨 | 大量花瓣从顶部飘落 | 3s | 物理模拟 |
| 落叶（分心标记） | 缓慢旋转下坠 + 左右飘荡 | 10s 后自动停止 | 物理模拟+`ease-out` |

### 8.3 反馈 300ms 原则

> 戒指事件 → 屏幕视觉回应 <300ms（PRD 冻结要求）

| 戒指事件 | 屏幕回应 | 目标延迟 |
|----------|---------|---------|
| 双击（专注开始） | 天空变暖金 + 光束出现 | <200ms |
| 双击（专注结束） | 结算动画（植物入册） | <300ms |
| 拍击（诚实标记） | 落叶飘下 + Nexi 回头看 | <200ms |
| wave（召唤 Nexi） | Nexi 走近动画 | <250ms |
| rotate_front（种新种子） | 种子落下动画 | <250ms |
| rotate_back（切换天气） | 天气渐变开始 | <200ms |

节奏敏感动画（呼吸引导灯环、月光起伏）由**前端本地驱动**，不经过云端。

#### 🥇 第一哇：晨间露珠

> 双击戒指"起床" → 花园从月夜渐变到晨曦 → 花朵上凝聚着晶莹露珠 → Nexi 揉着眼睛走出来说"你深睡只有 58 分钟..."

- **技术要求**：露珠 shader（球体+环境反射）+ 数量映射逻辑

#### 🥈 第二哇：诚实落叶

> 专注中 → 选手故意晃动戒指 → 生长暂停 → 选手拍击戒指诚实标记 → 一片叶子从植物上缓缓飘落 → 手机围观端同步看到 → Nexi 回头看（担忧但非责备）

- **评审心理**："不惩罚分心，只奖励诚实——这个设计好。"
- **截图价值**：⭐⭐⭐⭐
- **技术要求**：落叶物理模拟 + 围观端 WS 同步

#### 🥉 第三哇：月光呼吸

> 选手躺下/静止 → 屏幕渐暗进入月光模式 → 月光亮度随戒指 IMU 体动数据起伏 → 花园在"呼吸" → 萤火虫在月光中游走

- **评审心理**："花园是活的。它真的被戒指数据驱动。"
- **截图价值**：⭐⭐⭐⭐⭐
- **技术要求**：IMU 数据 → 月光强度映射 + 萤火虫粒子

## 11. 专注模式工作台：AI 规划 + 日程 + 待办

> **当用户双击戒指进入专注模式时，花园退为情绪背景，工作台浮为功能焦点。**

### 11.1 核心设计理念

| 对比维度 | 花园模式（休息/日常） | 专注模式工作台 |
|----------|---------------------|---------------|
| **花园角色** | 主角，全屏 | **情绪背景**：强模糊 + 暗角 |
| **信息密度** | 极低（无数字，无UI） | **高**：计时器、待办、日程 |
| **Nexi 角色** | 陪伴者（做自己的事） | **秘书/监工**：轻声提醒、鼓励 |
| **交互重心** | 戒指手势 + 视觉反馈 | **视觉信息** + AI 对话 |
| **色彩氛围** | 5 态渐变 | 统一暗化底层 + 高亮功能区 |

### 11.2 进入/退出专注模式的过渡动画

```
【进入专注】双击戒指
  ├─ 0.0s: 天空光束增强（金色丁达尔光束出现）
  ├─ 0.3s: 花园开始模糊 (CSS filter blur 0→16px, 1.5s)
  ├─ 0.5s: 暗角加深 (5%→35%)
  ├─ 0.5s: 工作台卡片从底部滑入 (translateY 100%→0, 1s)
  ├─ 0.8s: 聚焦计时器数字放大弹入 (scale 0.8→1.05→1)
  ├─ 1.0s: 待办列表项逐条淡入 (stagger 0.1s each)
  ├─ 1.2s: Nexi 缩小移至右下角，切换"静修"动作
  └─ 1.5s: 过渡完成，工作台就绪

【退出专注】双击戒指
  ├─ 0.0s: 结算动画触发（植物入册 + 专注时长展示）
  ├─ 0.5s: 工作台卡片向下滑出
  ├─ 0.8s: 花园解模糊 (blur 16px→0)
  ├─ 1.0s: 暗角恢复 (35%→5%)
  ├─ 1.0s: Nexi 回到正常位置/大小
  └─ 1.5s: 过渡完成，回归花园模式
```

### 11.8 AI 对话条（底部）

这是用户与 Nexi 进行**流式编排对话**的入口。

```
┌──────────────────────────────────────────────────────┐
│  💬 "论文写完后，要不要把数学复习提前到 16:00？          │
│      今晚早点睡，明天状态会好很多。"                     │
│                                                      │
│  ┌──────────────────────────────────────┐  ┌────┐   │
│  │  打字或语音说...                      │  │ 🎤 │   │
│  └──────────────────────────────────────┘  └────┘   │
│                                        ┌────┐       │
│                                        │Nexi│       │
│                                        └────┘       │
└──────────────────────────────────────────────────────┘
```

#### 流式对话的视觉表现

LLM 返回的文本是**逐字流式打印**的，而非一次性显示：

```
帧 1:  "论"
帧 2:  "论文写完"
帧 5:  "论文写完后，要不"
帧 15: "论文写完后，要不要把数学复习提前到 16:00？"
帧 20: "论文写完后，要不要把数学复习提前到 16:00？\n今晚早点睡，明天状态会好很多。"
```

| 参数 | 值 |
|------|-----|
| 打字速度 | ~15-20 字符/秒（模拟自然语速） |
| 光标 | 闪烁竖线 `│` 跟随机打在文本末尾 |
| 完成后 | 光标消失 → 气泡停留 5s → 自动收起或等待用户回复 |
| 多段落 | 段落间 300ms 停顿 + 换行 |

#### AI 对话的动作类型

Nexi 的流式对话不仅输出文本，还会**触发布局动作**：

| 对话意图 | 文本示例 | 触发的 UI 动作 |
|----------|---------|---------------|
| **新增待办** | "我帮你把'回复邮件'加到待办里了" | 待办列表新增一项，带高亮闪烁 |
| **调整日程** | "我把数学复习挪到 16:00 了" | 日程时间线对应项移动到新位置 |
| **状态提醒** | "你已经分心 3 次了，休息 5 分钟吧" | 聚焦计时器卡片边框变橙色脉冲 |
| **鼓励** | "还剩 12 分钟，你做得很好" | 计时器进度条闪烁金色 |
| **建议** | "要不要现在休息一下？" | 弹出 [休息 5 分钟] / [继续专注] 按钮 |
| **晨间规划** | "今天安排 3 件事：论文、复习、运动" | 待办清单自动填充 + 日程时间线自动生成 |

### 11.10 Nexi 在专注模式下的行为

专注模式下 Nexi **缩小并移至右下角**，角色从"陪伴者"切换为"监工"：

| Nexi 行为 | 触发条件 | 表现 |
|-----------|---------|------|
| **静修看书** | 专注进行中（默认） | 右下角小尺寸（50%），安静看书，偶尔抬头 |
| **轻声提醒** | 分心检测触发 | 抬头，对话气泡："已经 3 分钟了哦，需要帮忙吗？" |
| **鼓励** | 完成一个番茄钟 | 微笑 + 气泡："25 分钟！休息 5 分钟吧。" |
| **关切** | 用户拍击诚实标记 | 担忧表情 + 气泡："没关系，我们继续。" |
| **主动建议** | 连续分心 3 次 | "要不要把当前任务拆分一下？太大容易分心。" |
