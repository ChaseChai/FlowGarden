## 1. 产品概述

### 1.1 一句话产品

> **镜园是重建生活秩序的下一代 AI 生活规划师——它住在一座会呼吸的花园里，戴在你手上的戒指是它的眼睛。**
>
> Don't just survive the day. Own your next day. —— Nexday slogan（保留）

### 1.2 融合叙事：为什么是 Nexday × 镜园

Nexday 是一个"会主动找你、记得你、为你负责"的 AI 秘书，但它**没有眼睛**——它要知道"你周二下午效率低"，只能靠你告诉它。镜园的戒指让它第一次有了**真实的眼睛**：不用你说，它就懂。而镜园原本缺的，正是 Nexday 这颗"成长闭环大脑"。三者融合各取所长：

| 层 | 来自 | 作用 |
|----|------|------|
| **大脑** | Nexday | 成长闭环：长期记忆 + 主动干预 + 周期复盘（秘书人格） |
| **身体** | 镜园 | 数字人 Nexi（秘书人格化 + 你的状态映射，Nexday 无形象） |
| **世界** | 镜园 | 绚丽花园（状态艺术化 + 情绪价值载体，Nexday 只有空白数据区） |
| **眼睛** | 戒指 | 真实感知（Nexday 全靠对话自述，镜园用戒指补全） |

**关键升级**：Nexday 的"长期记忆"原本依赖对话自述，镜园让它落地为**戒指真实数据**——记忆不再是"你说你累了"，而是"戒指显示你昨晚深睡 58 分钟"。**数据让秘书的话第一次有根据。**

**答辩金句**：
> "Nexday 是一个会主动找你、记得你的秘书——但它没有眼睛，只能听你说。镜园给了它一双戒指做的眼睛，和一座它会住进去的花园。"

### 1.3 目标用户（继承 Nexday）

| 层级 | 画像 | 核心诉求 |
|------|------|----------|
| 核心层 | 大三考研/考公党（Nexday 用户故事 Alex 原型） | 自律难、心态易崩，需全程监督与陪伴 |
| 增长层 | 独居青年/年轻创作者 | 生活作息混乱，需生活节律重建 |
| 延展层 | ADHD/注意力困难群体 | 低压力、被理解的状态管理 |

### 1.4 成功指标（展演导向）

| 指标 | 目标 |
|------|------|
| 视觉冲击 | 评委 10 秒内"想截图"（月光花园 / 数字人打哈欠） |
| 展演核心闭环 | 现场跑通"戒指数据→花园呼吸 + 戒指手势→花园状态 + AI秘书对话"三轴 |
| 情绪价值 | ≥1 个让评委会心一笑的瞬间（数字人因你熬夜打哈欠） |
| 技术可信 | 戒指数据真实驱动（非纯动画），AI 秘书有记忆感 |
| 工期可控 | 2-3 天出可演示前端，7 天完整 |

## 2. 产品概念：三位一体

### 2.1 三层融合模型

```
现实世界（你）           心灵花园（世界层）          数字人 Nexi（身体层）
戒指感知 + 手势 + 语音  →  专注→生长  分心→落叶    →  你专注它静修
                          深睡→露珠  疲惫→黄昏        你熬夜它打哈欠
                                            你离开它想念
                                                ↓
                                    AI 秘书大脑（Nexday 大脑层）
                                    长期记忆 + 主动干预 + 周期复盘
```

### 2.2 Nexday 成长闭环如何被戒指"看见"

Nexday 原始 GROWTH LOOP：**长期记忆 → 关键时刻干预 → 周期复盘 → 迭代**。镜园的升级是把每一环的数据来源从"对话自述"换成"戒指真实感知"：

| 闭环环节 | Nexday 原始（靠说） | 镜园升级（戒指看见） |
|----------|---------------------|---------------------|
| 长期记忆·行为模式 | "你说你周二下午效率低" | 戒指记录的专注曲线统计 |
| 长期记忆·睡眠 | "你说你昨晚没睡好" | actigraphy 真实分期 |
| 主动干预·深夜刷手机 | 监测娱乐 App 使用 | 戒指检测该睡时段仍在活动 + 秘书温柔警示 |
| 主动干预·计划偏离 | 任务未完成 | 专注 session 中断/分心标记 |
| 周期复盘 | 自述总结 | 真实睡眠-专注关联数据周报 |

**这就是融合的命门：秘书的话第一次有根据。**

### 2.3 24 小时四时段陪伴（继承 Nexday 秘书人格）

| 时段 | 秘书行为 | 示例台词（数字人 Nexi 说） |
|------|----------|----------------------------|
| 清晨 | 晨间简报 | "早安。昨晚深睡只有 58 分钟——这就是你今天累的原因。我把难的挪到下午了，上午先回邮件吧。" |
| 上午 | 专注陪伴 | "接下来 45 分钟我陪你。我在看书，你专心。"（数字人在花园做自己的事） |
| 傍晚 | 晚间复盘 | "今天 4 件事完成，3 次分心都诚实记录了，很棒。明天的日程排好了，要看看吗？" |
| 睡前 | 入睡仪式 | "明天 8 点有会，我们 11 点睡好吗？现在跟我做三轮深呼吸……" |

### 2.4 设计原则（v3 优先级倒转）

1. **前端美学是工期重心**：2-3 天必须出"想截图"的画面；好看 > 技术生成（参考《旅行青蛙》《Gris》）。
2. **情绪价值优先**：数字人 + 花园是 Nexday 没有的差异化，是评委记住你的理由。
3. **Demo 不死**：真戒指 + 预录回放双通道；云端 + 本地双模式。
4. **滋养非惩罚**（反 Forest）：分心只暂停生长，诚实承认才有奖励性反馈。
5. **检测精度够用即可**：睡眠/专注判定只作"趋势指标"，不深究医学精度，不画大饼。
6. **戒指是哑终端**：所有智能在 bridge/云端/前端，戒指只提供原语。

## 3. 核心展演闭环（v3 重点：必须现场跑通的三轴）

展演的成败取决于三轴是否现场可见。这是开发优先级的唯一真相来源。

### 3.1 数据轴：戒指 IMU → 花园"活着"

戒指 25Hz 六轴数据实时驱动花园视觉，让花园"会呼吸"：
- **专注模式**：手部静止 → 种子破土生长；微动 → 生长进度推进；剧烈动 → 暂停。
- **睡眠/月光模式**：体动强度 → 月光亮度起伏（"花园随你的睡眠呼吸"）。
- **围观端**：评委手机同步看到这条"活"的曲线。

**这是"非纯动画"的技术可信度来源——花园是真的被你的手驱动的。**

### 3.2 手势轴：戒指手势 → 花园状态切换

冻结的交互映射表（D1 写入常量）：

| 模式＼原语 | 单击（拍击） | 双击 | wave | rotate_front | rotate_back |
|------------|--------------|------|------|--------------|-------------|
| **专注** | 标记分心（落叶+诚实记账） | 结束专注→结算 | 忽略 | 忽略 | 忽略 |
| **休息** | 击掌/触发分身小动作 | 进入专注 | 召唤分身走近 | 种新种子 | 切换天气 |
| **睡眠** | 忽略（防翻身误触） | 起床→晨间报告 | 忽略 | 忽略 | 忽略 |

**反馈 300ms 原则**：戒指无自带反馈，手势→屏幕视觉回应必须 <300ms。

### 3.3 秘书轴：AI 成长闭环（Nexday 灵魂）

现场可演示的秘书行为：
- **晨间简报**（双击起床触发）：睡眠数据 + 今日建议 + 数字人口播。
- **主动干预**（展演用预设场景）：该睡时段检测到活动 → Nexi 温柔警示"明天还有早课，现在的快乐是透支明天的精力哦"（Nexday 原话保留）。
- **晚间复盘**：今日完成/分心统计 + 明日规划对话。

三轴合一即为"戒指驱动花园 × AI 秘书规划"的完整展演。

## 4. 系统架构（精简）

### 4.1 四层

```
感知层（戒指哑终端）—BLE— 边缘层（ring_bridge：事件总线+双引擎+聚合）
                —WSS— 云端层（Zeabur：hub/db/asr/llm-proxy/web）
                —WSS/HTTPS— 表达层（Three.js 花园 + Live2D Nexi + 秘书对话 UI）
```

### 4.2 双模式（Demo 不死）

| 模式 | 开关 | 链路 | 用途 |
|------|------|------|------|
| 云模式 | `CLOUD=1` | bridge→Zeabur→公网前端 | 展演/围观/数据积累 |
| 本地回退 | `CLOUD=0` | bridge→localhost→本机前端 | 会场断网兜底 |
| 演示回放 | `DEMO_MODE=1` | 预录数据驱动全部状态机 | BLE 全挂兜底（本地/云端均可） |

### 4.3 技术选型

| 层 | 选择 | 理由 |
|----|------|------|
| 前端花园 | Three.js + 预渲染美术 + 粒子/光照层 | 粒子生态成熟，2-3 天可出氛围 |
| 数字人 | Live2D 首选（VRM 备选） | 轻量、表情丰富、加载快 |
| 动效 | gsap 2s 渐变 | 省心 |
| 边缘 | Python + bleak（SDK 零改动） | 现成 |
| 云端 | Zeabur：FastAPI hub + PostgreSQL + faster-whisper small + LLM 代理 | 一键部署 |
| 数据源 | `SensorSource` 协议（RingSource/DemoSource） | 硬件解耦，回放透明 |

## 5. 戒指交互规范（冻结，精简）

**六原语**：IMU 流（`0x0605`）/ 单击（`0x0704`）/ 双击（`0x0703`）/ 手势×3（`0x0702`：wave·rotate_front·rotate_back）/ 录音（`0x0505`）/ LED（固件固定，不入产品语义）。

**约束**：① IMU 需手势模式 + `start_sensor_report()` 成功（录音模式返错码2，引导用户单击切换）；② `receive_auto_audio_file` 不与 `download_audio_file` 并发；③ 断连指数退避重连≤5 次，断连段不伪造数据；④ 睡眠模式屏蔽手势/单击防误触。

**硬件哲学**：戒指只提供原语，所有智能在上游——换掉戒指、换手机 IMU 或预录数据，系统照跑。

### P0 详述（2-3 天必须完成）

**F2 绚丽花园前端**（工期第一重心）：
- Three.js + 预渲染美术 + 粒子层（花瓣/落叶/萤火虫/露珠/花粉）+ 光照层（晨光/正午/黄昏/月光）。
- 5 态状态机（深度心流/浅专注/分心/疲惫/休息睡眠）gsap 2s 渐变。
- 植物 4 档预制骨骼缩放（种子→芽→株→花）；稀有花独立贴图。
- **验收**：5 态可切换、过渡自然、画面"想截图"。

**F3 戒指驱动花园**（展演核心）：
- IMU 实时流 → 专注模式生长进度 + 月光模式亮度起伏。
- 手势/按键事件 → 18 格映射表状态切换，反馈 <300ms。
- **验收**：现场静止→发芽、晃动→暂停、拍击→落叶、wave→召唤 Nexi 全链路 <90s。

**F4 数字人 Nexi**：
- Live2D；8 动作：静修/修剪看书/回头看/打哈欠揉眼/击掌/跳舞/守夜/门口等待。
- 承载秘书"说话"（文字气泡默认 + TTS 可选），永不阻塞画面。
- **验收**：晨间报告由 Nexi "说出"；≥5 动作现场触发。

**F5/F6 AI 秘书**：
- 晨间简报：睡眠数据 + 今日建议 + Nexi 解读（LLM 生成，模板兜底）。
- 晚间复盘：今日完成/分心统计 + 明日规划对话。
- **验收**：双击起床到简报 ≤3s；LLM 不可用模板兜底。

### P1 简述

- **F7 睡眠趋势**：actigraphy 60s epoch，输出"深睡/浅睡/醒"趋势与翻身次数；**定位消费级趋势，不宣称医疗精度**。
- **F8 专注判定**：5s 滑窗方差+过零率，三档（深度心流/专注/分心）。
- **F9 主动干预**（Nexday 亮点展演）：预设场景演示——该睡时段检测活动 → Nexi 温柔警示；计划偏离 → "已帮你把任务移到今晚 8 点"。
- **F10 入睡仪式**：4-7-8 呼吸引导（前端本地驱动）+ 渐暗熄屏。
- **F11 语音对话**：录音→云 ASR→LLM 意图（对话/指令/情感）→Nexi 回应；降级预置指令。
- **F12 长期记忆**：戒指真实数据 + 对话上下文，构成 Nexday 全息记忆（行为模式/睡眠/情绪/关键事件）。
- **F14 云端围观**：二维码→手机只读花园（粒子降档）+ 喝彩按钮触发主屏花瓣雨。

## 7. 前端规范（v3 工期重心）

### 7.1 页面结构

```
/           主界面（花园 + Nexi + 模式状态条 + 连接指示 + 秘书对话气泡）
/report     晨间报告（分数 + 睡眠曲线 + Nexi 解读）
/review     晚间复盘（今日完成 + 分心回放 + 明日规划）
/watch      围观端（手机响应式，只读花园 + 喝彩）
/gallery    公开画廊（P2）
```

### 7.2 视觉风格

- **默认皮肤·水彩花园**：吉卜力/《Gris》柔和治愈，契合蓝盒子"休息"主题与灵光"Vibe"赛道。
- 备选皮肤（P2）：像素庭院（《星露谷》）、国风庭院（水墨留白，兼容"中国文化出海"）。
- 色彩系统：5 态各有主色（金/柔白/微暗雨/暖橙黄昏/蓝调月光），gsap 渐变。

### 7.3 花园场景（garden/）

- `scene.js`：场景图 + 5 态状态机。
- `plants.js`：4 档预制模型，生长值驱动骨骼缩放；稀有花独立贴图。
- `weather.js`：粒子池复用（≤5 种发射器），移动端粒子减半。
- `dew.js`：晨间露珠数量 ∝ 深睡趋势。
- `moonlight.js`：月光模式亮度随 IMU 体动起伏（数据轴可视化）。

### 7.4 数字人（avatar/）

- `avatar.js`：Live2D 加载渲染。
- `motions.js`：8 动作状态机，输入=全局状态+ring 事件，切换 300ms 淡入。
- `speech.js`：文字气泡（默认）+ TTS（可选），不阻塞画面。

### 7.5 性能预算

主屏 60fps（中端笔记本）；移动端 ≥30fps；WS 断线 5s 内提示并重连。

## 8. AI 秘书大脑规范（Nexday 融合核心）

### 8.1 三特性（继承 Nexday）

| 特性 | 实现 |
|------|------|
| 活的（Alive） | 主动发起：晨间简报、深夜警示、晚间复盘，非被动应答 |
| 记得一切 | 长期记忆 = 戒指真实数据 + 对话上下文（F12） |
| 为你负责 | 不只记录，推动执行闭环（干预 + 复盘迭代） |

### 8.2 长期记忆四维（Nexday 全息记忆，数据源升级）

| 维度 | Nexday 原始 | 镜园升级 |
|------|-------------|----------|
| 行为模式 | 对话自述 | 戒指专注曲线统计 |
| 睡眠 | 对话自述 | actigraphy 真实趋势 |
| 情绪周期 | 对话推断 | 语音情感 + 分心标记频次 |
| 人生关键事件 | 对话记录 | 对话记录（同） |

### 8.3 主动干预（展演用预设场景）

| 触发 | Nexi 行为（台词保留 Nexday 原味） |
|------|-----------------------------------|
| 该睡时段检测到活动 | "明天还有早课，现在的快乐是透支明天的精力哦。" |
| 专注 session 中断/计划偏离 | "已帮你把任务移到今晚 8 点，那时你通常比较空。" |
| 连续分心标记 | "这件事真的很重要，我们现在就做 5 分钟好吗？" |

### 8.4 LLM Prompt 设计（秘书人格）

```
你是 Nexi，住在用户花园里的 AI 生活秘书。你拥有长期记忆（用户睡眠/专注/情绪数据）。
原则：①滋养非惩罚，不说"你又失败了"，说"我们今天都有点累"；
②话有根据——引用真实数据（"你昨晚深睡只有58分钟"）；
③主动负责——推动执行而非仅记录。
当前数据：{sleep_summary, focus_today, distractions}
任务：{morning_briefing | evening_review | intervention}
```

模板兜底文案库：LLM 不可用时按数据槽位填模板。

## 9. 后端云（精简）

- **hub**（FastAPI+WS）：bridge 上行接入、事件广播、围观连接。
- **api**：REST 报告/趋势/画廊。
- **db**（PostgreSQL）：`sleep_epoch`/`focus_session`/`distraction_mark`/`plant`/`memory_event`。
- **asr**（faster-whisper small CPU）：`POST /api/asr` WAV→文本。
- **llm-proxy**：LLM/TTS 密钥代理 + 模板兜底。
- **web**：静态托管 + 公网域名 + 围观二维码。
- **隐私**：原始 IMU 边缘聚合，云端只存分钟级摘要；语音转写后即焚。
