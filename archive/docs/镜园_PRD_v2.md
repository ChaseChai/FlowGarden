## 1. 产品概述

### 1.1 一句话产品

> **镜园是一款状态感知的游戏化健康伴侣：一枚戒指读懂你的睡眠与专注，数据长成一座花园，数字分身与你同呼吸。**

场景化表达：**"一枚戒指 + 一座花园 + 一个另一个你"**。

### 1.2 定位与竞品锚点（设计依据）

镜园站在三个被验证过的市场交叉点，占据"数据真实度 × 表达温度"的右上空白象限：

| 竞品 | 验证了 | 它的软肋（=我们的设计原则来源） |
|------|--------|--------------------------------|
| **Forest**（月活峰值 1800 万） | 游戏化专注的千万级需求 | 手动打卡无数据壁垒；枯树惩罚制造负罪感 → 镜园**自动感知 + 滋养式设计** |
| **Focus Friend**（上线 1 月登顶美区） | "被督促者→帮助者"视角转换 | 无记录、无身体数据 → 镜园分身**映射真实数据** |
| **Oura Ring**（550 万枚，估值 110 亿美元） | 戒指是睡眠监测最优形态（PSG 一致性 79% vs 手腕 60-65%） | 冷冰冰分数报告、零游戏化 → 镜园**世界化表达**（深睡不是 58 分钟，是花瓣上的露珠） |
| **Endel**（月流水稳定 70 万美元） | 个性化状态管理是最强付费点 | → 镜园 F11 睡眠-专注关联洞察正属此类 |
| **Finch / Replika** | 情感陪伴粘性 | 与真实数据脱节 → 镜园分身**有数据的人格** |

**30 秒定位话术（答辩用）**：
> "Forest 种树，但树不认识你的身体；Oura 认识你的身体，但只会打 78 分；Finch 的宠物可爱，但不知道你昨晚睡没睡好。镜园是第一个把三者合起来的产品——**严肃的数据，温柔的表达**。"

### 1.3 目标用户

| 层级 | 画像 | 核心诉求 |
|------|------|----------|
| 核心层 | 18-25 岁学生/考研党/自习人群（Forest 存量用户） | 专注陪伴，拒绝惩罚式打卡 |
| 增长层 | 22-30 岁年轻职场人/创作者 | 搞懂"为什么睡 8 小时还累" |
| 延展层 | ADHD/注意力困难群体 | 低压力、被理解的状态管理 |

## 2. 产品概念模型

### 2.1 三层映射世界观

```
现实世界（你）                心灵花园（映射层）            数字分身（人格层）
动作/手势/语音/睡眠      →    专注→生长  分心→落叶      →    你专注它静修
                             深睡→露珠  疲惫→黄昏            你熬夜它打哈欠
                                                       你离开它想念
```

### 2.2 24 小时产品节律

| 时段 | 模式 | 核心功能 | 花园状态 | 数字人状态 |
|------|------|----------|----------|------------|
| 清晨 | 晨间简报 | 睡眠报告 + 今日建议 | 花朵带露珠（量 ∝ 深睡比例） | 同步你的精神状态 |
| 白天 | 专注模式 | 专注判定、生长/落叶 | 随专注生长、随分心落叶 | 静修/修剪/陪伴 |
| 白天 | 休息模式 | 手势互动、语音对话 | 月光模式 | 苏醒、互动 |
| 睡前 | 入睡仪式 | 呼吸引引、渐暗熄灭 | 夜色加深 | 引导呼吸 |
| 夜间 | 睡眠监测 | actigraphy 分期 | 月光呼吸（随体动起伏） | 守夜 |

### 2.3 设计原则（全员对齐，优先级从高到低）

1. **Demo 不死是第一原则**：真戒指 + 预录回放双通道；云端 + 本地双模式；任一外部依赖有降级。
2. **惩罚最小化（反 Forest）**：检测到的分心只暂停生长；只有"诚实承认"才有视觉反馈，且反馈是奖励性的（诚实之泉成就）。
3. **戒指是哑终端**：感知智能在算法层，表达温度在花园层；戒指只提供原语，不背任何"智能"包袱。
4. **反馈 300ms 原则**：戒指无任何自带确认反馈（无震动/LED 不可编程），戒指事件 → 屏幕视觉回应必须 <300ms。
5. **严肃的数据，温柔的表达**：底层医学标准 actigraphy（Oura 同源），表达层世界化；不宣称医疗级精度，诚实即差异化。
6. **滋养而非考核**：数字人永远不说"你又失败了"，而是"我们今天都有点累，早点休息吧"。

## 3. 系统架构总览

### 3.1 四层架构

```
┌─ 感知层（哑终端）─────────────────────────────┐
│ 语音戒指：IMU 0x0605 / 按键 0x0703·0x0704 /      │
│ 手势 0x0702 / 录音 0x0505 / LED（固件固定）      │
└──────────────────┬─────────────────────────────┘
                   │ BLE（NUS）
┌─ 边缘层（现场笔记本，必须本地）─────────────────┐
│ ring_bridge.py（Python + ring_sound SDK）        │
│ · 事件总线 RingEvent                            │
│ · focus_detector.py 专注判定（5s 滑窗）          │
│ · actigraphy.py 睡眠分期（60s epoch）            │
│ · 体动边缘聚合：原始流 → 分钟级活动计数           │
│ · demo_source.py 预录回放（DEMO_MODE）           │
└──────────────────┬─────────────────────────────┘
                   │ WSS 上行（CLOUD=1）/ 本地 WS（CLOUD=0）
┌─ 云端层（Zeabur）──────────────────────────────┐
│ hub：WS Hub（事件广播 / 状态同步）                │
│ api：REST（报告 / 趋势 / 画廊）                   │
│ db：PostgreSQL（睡眠 epoch / 专注 session / 事件）│
│ asr：faster-whisper small（CPU，10s 语音 ≈1-3s） │
│ llm-proxy：LLM/TTS API 密钥代理 + 模板兜底        │
│ web：前端静态托管（公网 URL + 围观二维码）         │
└──────────────────┬─────────────────────────────┘
                   │ WSS / HTTPS
┌─ 表达层（任意设备浏览器）───────────────────────┐
│ 主屏：Three.js 花园 + Live2D 数字人 + 报告 UI     │
│ 围观端：手机响应式花园（粒子降档，只读 + 喝彩互动） │
└─────────────────────────────────────────────────┘
```

**分工哲学**：戒指干"感知的体力活"，边缘干"实时的算法活"，云端干"存储与 AI 的重活"，前端干"表达的细活"。**换掉任何一个环节，其他三层不受影响**（硬件可迁移 / 云端可关闭 / 前端可降级）。

### 3.2 双模式部署（环境变量一键切换）

| 模式 | 开关 | 链路 | 用途 |
|------|------|------|------|
| 云模式（默认） | `CLOUD=1` | bridge → Zeabur Hub → 公网前端 | 演示/围观/数据积累 |
| 本地回退 | `CLOUD=0` | bridge → localhost WS → 本机前端 | 会场断网兜底 |
| 演示回放 | `DEMO_MODE=1` | 预录数据驱动全部状态机 | BLE 全挂时兜底（本地/云端均可回放） |

### 3.3 技术选型总表

| 层 | 决策 | 选择 | 理由 |
|----|------|------|------|
| 边缘 | 桥接 | Python + bleak（ring_sound SDK 零改动） | SDK 现成，团队熟悉 |
| 边缘 | 数据源抽象 | `SensorSource` 协议（RingSource / DemoSource） | 硬件解耦，回放透明 |
| 云端 | WS Hub | FastAPI + WebSocket（或 Node ws） | 与边缘同语言，~100 行可落地 |
| 云端 | 数据库 | PostgreSQL（Zeabur 一键） | 跨天历史 + 关联查询 |
| 云端 | ASR | faster-whisper small（CPU int8） | 免费可控，短指令延迟可接受；API 为备选 |
| 云端 | LLM/TTS | 外部 API + 代理层 + 模板兜底 | 现场网络不可控，必须有离线降级 |
| 前端 | 花园 | Three.js + 预渲染美术 + 粒子/光照层 | 粒子/光照生态成熟 |
| 前端 | 数字人 | Live2D 优先（VRM 备选） | 轻量、表情丰富、48h 可落地 |
| 前端 | 动效 | gsap 状态过渡（2s 渐变） | 成熟省心 |

### 3.4 通信协议（冻结）

**WS 消息（bridge → hub → 前端）**：

```json
// 状态引擎
{"type": "state", "value": "deep_focus|focus|distracted|rest|sleeping", "ts": 123}
// 戒指原始事件（前端互动响应用，优先级最高）
{"type": "ring", "event": "tap|double_tap|gesture:wave|gesture:rotate_front|gesture:rotate_back"}
// 睡眠报告
{"type": "sleep_report", "score": 72, "deep_min": 58, "awakenings": 1, "cycles": 4, "stages": [...]}
// 实时体动（月光呼吸/围观可视化用，已聚合为每分钟活动计数）
{"type": "activity", "epoch": 1234560, "count": 3}
// 语音管线结果
{"type": "voice", "text": "识别文本", "intent": "chat|command|emotion", "reply": "分身回应"}
```

**REST（hub 提供）**：`GET /api/report/latest`、`GET /api/sleep/week`、`GET /api/focus/week`、`GET /api/gallery`（公开画廊）、`POST /api/asr`（WAV → 文本）。

**延迟预算**：ring 事件广播优先级最高，端到端（戒指→屏幕动画）目标 <300ms；云端中转容忍 50-200ms，节奏敏感动画（呼吸引导灯环）由前端本地驱动、云端只发起止令。

## 4. 戒指交互规范（冻结，D1 写入代码常量）

### 4.1 硬件哲学：戒指是哑终端

戒指 = **6 原语输入器 + 1 盏不可编程的灯**。它不能震动、不能发声、LED 不可编程、模式不可程序切换、不跑任何算法。所有智能都在 bridge / 云端 / 前端。

**答辩话术**：
> "戒指在我们系统里只是一枚传感器和开关——刻意的。感知的智能在算法层，表达的温度在花园里。硬件越简单，系统越可迁移：明天换任何一个带 IMU 的穿戴设备，镜园照样活着。"

### 4.2 六原语能力表

| 原语 | 协议/SDK | 数据特征 | 系统语义 |
|------|----------|----------|----------|
| IMU 六轴流 | `0x0605`（`start_sensor_report`/`wait_sensor_data`） | 25Hz 批量，每批 ≤32 采样 | 唯一连续数据源：专注判定、睡眠分期原料 |
| 单击（拍击） | `0x0704`（`wait_sensor_key_single_press_event`） | 事件，~500ms 双击窗口后到达 | 诚实标记分心 / 击掌互动 |
| 双击 | `0x0703`（`wait_sensor_key_double_press_event`） | 事件，排他 | 模式开关（专注开始/结束、起床） |
| 手势×3 | `0x0702`（`wait_sensor_gesture_event`） | wave / rotate_front / rotate_back | 休息模式互动 |
| 录音 | `0x0505` 自动上报（`receive_auto_audio_file`） | 长按触发，Speex→WAV | 语音对话原料（P1） |
| LED | 固件固定（绿=录音中，红=手势中，红闪=失败） | 不可编程 | 仅作底层状态提示，不进入产品语义 |

### 4.3 交互映射表（6 原语 × 3 模式 = 18 格，冻结）

| 模式＼原语 | 单击（0x0704） | 双击（0x0703） | wave | rotate_front | rotate_back |
|------------|----------------|----------------|------|--------------|-------------|
| **专注** | 标记分心（落叶+诚实记账） | 结束专注 → 结算 | — 忽略 | — 忽略 | — 忽略 |
| **休息** | 击掌/触发分身小动作 | 进入专注 | 召唤分身走近 | 种新种子 | 切换天气 |
| **睡眠** | — 忽略（防翻身误触） | 起床 → 晨间报告 | — 忽略 | — 忽略 | — 忽略 |

**规则**：睡眠模式在应用层屏蔽手势与单击；任何模式下未被映射的事件一律忽略，不产生副作用；此表 D1 定稿，之后只调表现、不改语义。

### 4.4 交互约束

1. **反馈 300ms 原则**：ring 事件 → 屏幕视觉回应 <300ms（事件消息在 WS 中走最高优先级通道）。
2. **模式前置条件**：IMU 实时流需戒指处于手势模式且 `start_sensor_report()` 成功（录音模式返回错误码 2 → `DeviceError`）。开机默认录音模式，首次使用引导用户单击切换；bridge 以软状态机记账并用 `0x0601` 探测纠错。
3. **并发互斥**：`receive_auto_audio_file()` 不与 `download_audio_file()` 并发（共享 `0x0505` 队列）；同一连接同一时间只发一个"命令-响应"请求，事件监听可并行。
4. **断连恢复**：指数退避重连（最多 5 次），断连段睡眠曲线标记"未知段"，不伪造数据；BLE 断开设备自动停 `0x0605` 上报，重连后需重发 `0x0601`。

### 5.2 P0 功能详述

#### F1 戒指桥接与数据接入

- **输入**：戒指 MAC（配置项）+ SDK 能力（`RingSoundClient`、`wait_sensor_data`、`wait_sensor_key_single_press_event`、`wait_sensor_key_double_press_event`、`wait_sensor_gesture_event`）。
- **处理**：扫描/直连 → 订阅 IMU 流与事件 → 心跳与指数退避重连（≤5 次）→ 边缘聚合（IMU 原始流 → 分钟级活动计数）→ 按 `CLOUD` 开关上行。
- **输出**：统一事件总线 `RingEvent = {type: imu_batch | tap | double_tap | gesture | disconnect, payload, timestamp}`。
- **数据源抽象**：`SensorSource` 协议（`RingSource` 真戒指 / `DemoSource` 预录回放），上层引擎无感知。
- **验收**：冷启动到首个 IMU 包 ≤10s；断连 30s 内自动恢复，UI 有连接指示；`DEMO_MODE=1` 下全流程可跑。

#### F2 睡眠监测与分期

医学经典 actigraphy（60s epoch）：

```python
activity = 该 epoch 内 |accel| 超阈值的采样点数
activity == 0      → deep_sleep
1 <= activity <= 2 → light_sleep
3 <= activity <= 8 → rem        # 手指微动——戒指独特优势
activity > 8       → awake
```

- **衍生指标**：总时长、入睡耗时、深睡比例、周期数（≈90min）、翻身次数、夜醒次数。
- **诚实边界**：消费级趋势参考，不做医疗级声明；分期逻辑现场可讲解、可回放验证。
- **验收**：预录一晚数据输出分期序列与指标；3 分钟模拟数据可演示分期变化。

#### F3 晨间睡眠报告

- **触发**：睡眠模式双击"起床"。
- **内容**：① 总结分（深睡 40% + 时长 30% + 周期 30%）② 指标卡片 ③ 睡眠结构时间轴 ④ 数字人一句话解读（LLM 生成，模板兜底）："你睡了 7 小时 42 分，深睡只有 58 分钟——比上周平均少 20%。这就是你今天累的原因。"
- **花园联动**：露珠数量/大小 ∝ 深睡比例。
- **验收**：双击到完整报告 ≤3s；LLM 不可用自动模板。

#### F4 专注模式

- **判定**（5s 滑窗，1s 步长）：`variance < T1 且 zero_crossing < T2 → deep_focus`（稀有花进度+2）；`variance < T1 → focus`（普通生长）；否则 `distracted`（仅暂停生长，不落叶）。
- **拍击 = 诚实记账**：落叶动画 + 记录分心时刻（时间戳 + 当时长）+ 成就进度。**惩罚最小化**（设计原则 2）。
- **双击 = 结束专注** → 结算（时长、深度比例、植物入册）。
- **验收**：现场"静止 30s 发芽 → 晃动 10s 暂停 → 拍击落叶"全链路 ≤90s。

#### F5 花园渲染与状态机

- **方案**：Three.js + 预渲染美术 + 粒子层（花瓣/落叶/萤火虫/露珠）+ 光照层（晨光/正午/黄昏/月光）；植物 4 档预制骨骼缩放动画；天气 4 态 gsap 2s 渐变。**不做算法生成，好看 > 技术**（参考《旅行青蛙》《Gris》治愈系逻辑）。

| 状态 | 天气 | 光照 | 粒子 |
|------|------|------|------|
| 深度心流 | 晴 | 金色暖阳 | 金色花粉 |
| 浅度专注 | 多云 | 柔白 | 微风飘絮 |
| 分心标记 | 雨（10s 短促） | 微暗 | 落叶飘零 |
| 疲惫 | 黄昏 | 暖橙 | 长影 |
| 休息/睡眠 | 月夜 | 蓝调月光 | 萤火虫/露珠 |

- **验收**：5 态现场可切换、过渡自然；画面达"想截图"水准。

#### F6 数字分身

- **形象**：Live2D 首选（VRM 备选）。
- **P0 动作集（8 个）**：静修冥想（专注中）/ 修剪看书（浅专注）/ 回头看（分心）/ 打哈欠揉眼（睡眠不足）/ 击掌打招呼（拍击）/ 跳舞庆祝（成就）/ 守夜坐姿（睡眠监测）/ 门口等待（≥3 天未互动）。
- **验收**：晨间报告由分身"说出"（TTS 或文字气泡）；≥5 动作现场可触发。

### 5.3 P1 功能（简版）

- **F7 休息互动**：按 18 格映射表执行（wave 召唤 / rotate_front 种植 / rotate_back 天气 / 拍击击掌）。
- **F8 入睡仪式**：屏幕 5min 渐暗 + 分身 4-7-8 呼吸引导（4s 扩张/7s 停留/8s 收缩，前端本地驱动）；IMU 检测渐静止 → 自动熄屏进 F2。
- **F9 语音对话**：录音 → `save_audio_bundle` → `POST /api/asr` → LLM 意图分类（对话/指令/情感）→ 分身回应；降级为预置指令菜单。
- **F10 数据页**：心流日/周热力图、睡眠周趋势、分心时刻回放列表。
- **F11 关联洞察**：`corr(昨晚深睡, 今日分心)` 周报卡片；**依赖云端历史数据，赛前真实佩戴积累**。
- **F12 云端围观**：演示页生成二维码；手机端只读花园（粒子降档）+ "喝彩"按钮（主屏飘落花瓣雨）。评委手机上"这座花园此刻是活的"。

### 5.4 P2 功能

- **F13 公开画廊**：`/gallery` 公网页，植物图鉴 + 匿名睡眠分布；小红书/B站传播素材。
- **F14 成就**：诚实之泉（10 次诚实标记）、极光之夜（连续 7 天深度专注 ≥2h）、月度织锦回顾。
- **F15 皮肤**：水彩（默认）/像素/国风 3 套资源切换（国风兼容"中国文化出海"叙事）。

## 6. 前端规范

### 6.1 页面结构

```
/           主界面（花园 + 数字人 + 模式状态条 + 连接指示）
/report     晨间报告（分数 + 曲线 + 分身解读气泡）
/data       数据页（心流热力 / 睡眠趋势 / 分心回放）
/watch      围观端（手机响应式，只读花园 + 喝彩）
/gallery    公开画廊（P2）
```

### 6.2 花园场景（garden/）

- `scene.js`：场景图 + 5 态状态机（天气/光照/粒子一键切换，gsap 2s 渐变）。
- `plants.js`：种子→芽→株→花 4 档预制模型/贴图，生长值驱动骨骼缩放；稀有花（deep_focus 积累）独立贴图。
- `weather.js`：粒子池复用（花瓣/落叶/萤火虫/露珠/花粉 ≤5 种发射器），移动端粒子数减半。
- `dew.js`：晨间露珠，数量 ∝ 深睡比例（F3 联动）。

### 6.3 数字人（avatar/）

- `avatar.js`：Live2D SDK 加载与渲染（备选 three-vrm）。
- `motions.js`：8 动作状态机，输入 = 全局状态 + ring 事件；动作切换带 300ms 淡入。
- `speech.js`：文字气泡（默认）+ TTS 音频（可选）二选一，永不阻塞画面。

### 6.4 UI 与响应式

- `report.js`：报告卡片 + 睡眠结构时间轴（Canvas 绘制，3s 内渲染完成）。
- `dashboard.js`：热力图/趋势图（轻量图表库或手绘 Canvas）。
- 围观端断点：≤768px 单列布局，隐藏数据页，只保留花园 + 喝彩按钮。
- **性能预算**：主屏 60fps（中端笔记本）；移动端 ≥30fps；WS 断线 UI 5s 内提示，自动重连。

### 6.5 主状态机（app.js）

```
MODES = focus | rest | sleep
事件源优先级：ring 事件（300ms 内必须视觉响应）> state 引擎 > 定时驱动（光照/呼吸）
任何未映射事件 → 忽略（18 格表为唯一真相）
```

## 7. 后端与云规范（Zeabur）

### 7.1 服务划分

| 服务 | 技术 | 职责 | 资源画像 |
|------|------|------|----------|
| hub | FastAPI + WS | bridge 上行接入、事件广播、状态同步、围观连接管理 | 常驻小内存 |
| api | FastAPI | REST（报告/趋势/画廊）、鉴权（可选 token） | 与 hub 同服务可合并 |
| db | PostgreSQL | 跨天历史持久化 | Zeabur 一键部署 |
| asr | faster-whisper small (int8) | `POST /api/asr`：WAV → 文本 | CPU 峰值型，冷启动可接受 |
| llm-proxy | FastAPI | LLM/TTS 密钥代理、模板兜底文案库 | 轻量 |
| web | 静态托管 | 前端构建产物 + 公网域名 | CDN 友好 |

### 7.2 数据模型（最小集）

```sql
sleep_epoch(night_id, epoch_ts, activity_count, stage)   -- 分钟级摘要，非原始流
focus_session(id, start_ts, end_ts, deep_ratio, plant_id)
distraction_mark(id, session_id, ts, at_duration)        -- 诚实记账
ring_event(id, ts, event, mode)                          -- 可选，调试回放用
plant(id, kind, rarity, born_session_id, created_ts)     -- 画廊素材
```

**隐私设计（答辩表述）**：体动原始数据只在边缘聚合，云端只存分钟级摘要；语音转写后即焚，不留音频；画廊默认匿名。睡眠是人最没有防备的状态，数据最小化是原则不是妥协。

### 7.3 部署与配置

- 环境变量：`DATABASE_URL`、`LLM_API_KEY`、`ASR_MODEL_SIZE=small`、`ALLOWED_ORIGINS`、`DEMO_MODE`。
- D1 当天完成 hub + web 部署拿到公网地址，之后每天开发即"线上"。
- 前端 WS 地址按 `CLOUD` 开关指向 wss://hub 或 ws://localhost。

## 10. 风险与对策

| 风险 | 等级 | 对策 |
|------|------|------|
| 现场 BLE 环境差连不上戒指 | **高** | DEMO_MODE 预录回放（本地/云端双份）；提前录全链路视频 |
| 会场 Wi-Fi 不稳云端失联 | **高** | `CLOUD=0` 本地回退模式，双模式都彩排 |
| IMU 需手势模式才能实时上报 | 中 | 彩排确认戒指模式；现场脚本写明"先单击切模式"；bridge 探测纠错 |
| LLM/ASR 现场无网 | 中 | 模板文案库兜底；语音降级预置指令 |
| 美术资源工期不足 | 中 | 先占位色块+粒子保证氛围，皮肤后置 |
| 云端延迟影响互动反馈 | 中 | ring 事件最高优先级通道；节奏敏感动画前端本地驱动 |
| 睡眠分期现场无法真验 | 低 | 预录真实整夜数据回放 + 算法逻辑讲解，坦白即加分 |
| 戒指续航/电量 <20% 拒绝操作 | 低 | Demo 前充满电；单次演示 <5min |
